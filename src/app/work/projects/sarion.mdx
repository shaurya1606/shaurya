## Overview

Sarion is a multi-drone AI disaster response platform designed to automate survivor detection, coordinate drone fleets, and simulate rescue missions in real time. The system integrates computer vision, autonomous navigation, and distributed mission coordination into a unified control dashboard.

The platform processes live video feeds using a trained YOLO model, detects survivors, logs geospatial coordinates, and deploys delivery drones using optimized routing algorithms. It also includes an offline simulation engine capable of generating mission replay videos using A* pathfinding and multi-agent coordination.

---

## Problem Statement

Disaster response operations often rely on manual scouting and uncoordinated rescue workflows, resulting in delayed survivor identification and inefficient resource allocation.

Existing limitations include:
- manual aerial monitoring
- lack of autonomous coordination
- inefficient routing of rescue drones
- absence of real-time telemetry dashboards
- no offline mission simulation capability
- redundant detection and duplicate data

Sarion addresses these issues through AI-driven detection and autonomous drone coordination.

## System Design

Sarion follows a layered architecture separating detection, orchestration, and simulation components.

```
Camera Input
      ↓
YOLO Detection Pipeline
      ↓
Mission Coordinator
      ↓
Drone Control Layer
      ↓
Routing Algorithms (TSP / A*)
      ↓
Database & Telemetry
      ↓
FastAPI Backend
      ↓
Dashboard UI
```

**Architecture Overview**

```
User Dashboard
      ↓
FastAPI Backend
      ↓
Mission Coordinator
      ├── Video Detection Service
      ├── Drone Simulation Engine
      ├── Route Planner
      └── Database Layer
             ↓
Drone Fleet (Scout + Delivery)
```

**Technology Stack**
- Backend: Python, FastAPI, Uvicorn, SQLite
- AI & CV: YOLO (PyTorch), OpenCV, NumPy
- Algorithms: A* search, Travelling Salesman Problem (TSP), spatial deduplication
- Simulation: Multi-threaded physics loop, grid-based pathfinding, video rendering pipeline

**Core AI Detection Pipeline**

```
Video Stream
      ↓
Frame Reader Thread
      ↓
YOLO Inference
      ↓
Person Detection
      ↓
Confidence Filtering
      ↓
GPS Simulation
      ↓
Database Logging
      ↓
Dashboard Update
```

The multi-threaded pipeline eliminates blocking operations and enables real-time inference.

**Drone Coordination Pipeline**

```
Detected Survivors
      ↓
Fetch Survivor Coordinates
      ↓
TSP Route Optimization
      ↓
Deploy Delivery Drone
      ↓
Goto Location
      ↓
Drop Kit Simulation
      ↓
Mark Delivered
```

**Simulation Engine Flow**

```
Upload Aerial Image
      ↓
YOLO Detection
      ↓
Generate Coverage Path
      ↓
Spawn Delivery Drones
      ↓
A* Pathfinding
      ↓
Delivery Execution
      ↓
Video Rendering
```

**Database Design**
- Survivor: id, lat, lon, confidence, status, detected_at
- MissionLog: id, timestamp, level, message, drone_id

**API Architecture**
- /api/video_feed
- /api/status
- /api/mission/start_scan
- /api/mission/stop_scan
- /api/mission/deploy_delivery
- /api/logs
- /api/settings
- /api/simulation/run

**Performance Optimizations**
- Multi-threaded detection pipeline
- Spatial deduplication of survivors
- Asynchronous telemetry polling
- Background drone physics simulation
- Efficient A* grid pathfinding
- YOLO confidence threshold filtering
