## Overview

LetsKraack is a distributed AI coaching assistant that unifies AI-generated courses, voice-based mock interviews, and resume evaluation in one workspace. It runs on a full-stack Next.js app with a serverless PostgreSQL backend and modular services.

---

## Objectives

- Unified AI coaching workspace
- Structured AI course generation
- Real-time voice interview simulation
- Job-specific resume analysis
- Stateless, scalable APIs and data persistence

---

## Architecture

```
Client UI
   ↓
Next.js App Router
   ↓
API Route Handlers
   ↓
Service Layer (AI / Voice / Resume)
   ↓
Neon PostgreSQL
```

Design principles: stateless REST APIs; modular services; parallel AI processing; persistent conversation tracking; context-based state management.

**Technology Stack**
- Frontend: Next.js 15, React 18, TypeScript, Tailwind CSS
- Backend: Next.js Route Handlers, Drizzle ORM, Neon PostgreSQL
- AI & Voice: Google Gemini, AssemblyAI (STT), AWS Polly (TTS)
- Infra/UX: Clerk auth, Radix UI, Framer Motion, React Hook Form

---

## AI Pipelines

**Course Generation**
```
User Input → Gemini Layout → Store Metadata → Parallel Chapter Generation → Fetch YouTube Videos → Save Content
```
Parallel generation reduces multi-chapter latency by ~40%.

**Voice Interview**
```
User Speech → AssemblyAI STT → Sliding Window (n=4) → Gemini Response → AWS Polly TTS → Playback
```
Delivers 500–2000 ms end-to-end latency.

**Resume Analysis**
```
Resume Upload → Job Context → Structured AI Evaluation → Feedback Persistence
```

## Challenges & Solutions

Navigating the intersection of real-time voice and high-volume AI generation presented several technical hurdles.

#### AI JSON Parsing
* **The Problem:** Gemini responses often included unwanted Markdown formatting (like ` ```json ` fences), causing `JSON.parse()` to fail.
* **The Solution:** Implemented a robust utility to strip markdown fences and sanitize the raw string before parsing it into the application state.

#### Latency in Course Generation
* **The Problem:** Generating a full course with 10+ chapters sequentially took upwards of 30 seconds, leading to poor UX.
* **The Solution:** Developed a parallelized generation pipeline using `Promise.all()` to trigger chapter content generation and YouTube video fetching concurrently.

#### Voice Synchronization
* **The Problem:** Significant audio lag occurred between user speech and AI playback, breaking the immersion of a "real" interview.
* **The Solution:** Switched to streaming transcription with AssemblyAI and optimized Gemini prompts to generate concise, high-impact responses for faster TTS conversion.

#### Token Usage
* **The Problem:** As mock interviews progressed, the conversation context grew too large, increasing both latency and API costs.
* **The Solution:** Implemented a **Sliding Window Memory** architecture that only retains and sends the last four messages to the AI, maintaining relevance while capping token overhead.
